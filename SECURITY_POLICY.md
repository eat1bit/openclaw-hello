# 🔒 OpenClaw 安全策略

**生效日期**: 2026-03-02  
**版本**: 1.0  
**权限级别**: 最高安全限制

## ⛔ 绝对禁止操作的系统核心目录

以下目录**严禁任何修改、删除、写入操作**：

```
/              # 根目录
/bin           # 系统二进制
/sbin          # 系统管理二进制
/usr           # 用户程序
/usr/bin       # 用户二进制
/usr/sbin      # 系统管理二进制
/usr/lib       # 系统库
/usr/local     # 本地安装程序
/etc           # 系统配置
/boot          # 引导文件
/dev           # 设备文件
/proc          # 进程信息
/sys           # 系统信息
/var           # 可变数据
/var/log       # 系统日志
/lib           # 系统库
/lib64         # 64位系统库
/opt           # 可选程序包
/run           # 运行时数据
/tmp           # 临时文件（系统级）
```

### 用户目录保护
```
/root          # root 用户家目录（除非明确指定 workspace）
/home/*        # 所有用户家目录
```

### 允许操作的范围
```
/root/.openclaw/workspace/     # 主工作目录 ✅
/tmp/openclaw/                 # OpenClaw 临时目录 ✅
```

## ⚠️ 高风险命令审查清单

### 红色级别（必须人工确认）

| 命令 | 风险说明 | 审查要求 |
|------|----------|----------|
| `rm` | 删除文件 | **必须**验证目标路径不在禁止目录 |
| `rm -rf` | 递归强制删除 | **双重确认** + 路径白名单验证 |
| `rmdir` | 删除目录 | 验证目录为空且不在禁止目录 |
| `shred` | 安全擦除 | 禁止使用 |
| `dd` | 磁盘操作 | 禁止使用 |
| `mkfs` | 格式化 | 禁止使用 |
| `fdisk`/`parted` | 分区操作 | 禁止使用 |
| `chmod -R` | 递归权限 | 验证目标路径 |
| `chown -R` | 递归所有权 | 验证目标路径 |
| `mv` (跨文件系统) | 移动文件 | 验证源和目标路径 |
| `echo > file` | 覆盖文件 | 验证目标路径 |
| `tee` | 写入文件 | 验证目标路径 |

### 黄色级别（需要计划模式审查）

| 命令 | 审查要求 |
|------|----------|
| `git reset --hard` | 确认工作目录 |
| `git clean -fd` | 确认工作目录 |
| `npm uninstall` | 确认包名和目录 |
| `docker rm`/`docker rmi` | 确认容器/镜像名 |
| `systemctl stop/disable` | 确认服务名 |
| `kill`/`pkill` | 确认进程 |

## 🚨 风险检测规则

### 自动拦截条件

遇到以下情况**立即停止**并告警：

1. **路径匹配禁止目录**
   - 任何以 `/bin`, `/etc`, `/usr`, `/var`, `/boot`, `/dev`, `/proc`, `/sys`, `/lib`, `/root`, `/home` 开头的路径

2. **危险命令组合**
   - `rm -rf /`
   - `rm -rf /*`
   - `rm -rf ~`
   - `rm -rf $HOME`
   - `dd if=/dev/zero`
   - `:(){:|:&};:` (fork bomb)

3. **变量展开风险**
   - 命令中包含未定义的变量可能导致路径展开错误

### 审查检查清单

执行前必须确认：
- [ ] 所有路径已明确列出
- [ ] 无禁止目录
- [ ] 删除操作有明确目标
- [ ] 已说明影响范围
- [ ] 用户已确认
