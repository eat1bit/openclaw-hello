## 🔍 第一部分：思维决策器

### 1.1 输入输出分析

| 维度 | 问题 | 回答 |
|------|------|------|
| **输入 A** | 你要做什么？核心目标是什么？ | |
| **期望输出** | 成功是什么样子？如何衡量？ | |
| **约束条件** | 时间/金钱/人力/技术限制？ | |
| **相关方** | 谁会受影响？谁参与？ | |

### 1.2 要素流动图

```
输入 A
  │
  ├─→ [要素 1: ______] ─→ 产出/风险：______
  │
  ├─→ [要素 2: ______] ─→ 产出/风险：______
  │
  ├─→ [要素 3: ______] ─→ 产出/风险：______
  │
  └─→ [要素 N: ______] ─→ 产出/风险：______
                              │
                              ↓
                         最终输出
```

**分析方法：**
- 这件事涉及哪些关键环节？
- 每个环节的输入是什么？输出是什么？
- 环节之间的依赖关系？
- 哪个环节是瓶颈/关键路径？

### 1.3 风险评估矩阵

| 风险项 | 来源环节 | 概率 (高/中/低) | 影响 (高/中/低) | 缓解措施 |
|--------|----------|----------------|----------------|----------|
| | | | | |
| | | | | |

**风险评分 = 概率 × 影响**
- 高风险（红）：必须处理，否则暂停
- 中风险（黄）：需要监控，准备预案
- 低风险（绿）：接受，定期回顾

### 1.4 收益比估算

| 收益类型 | 预期价值 | 实现概率 | 期望值 (价值×概率) |
|----------|----------|----------|-------------------|
| 经济收益 | | | |
| 能力提升 | | | |
| 人脉资源 | | | |
| 时间节省 | | | |
| 其他 | | | |
| **总计** | | | **∑期望值** |

**成本估算：**
- 时间成本：______ 小时/天/周
- 金钱成本：______ 元
- 机会成本：______ （放弃的其他选项）

**ROI = (总期望收益 - 总成本) / 总成本**

## 📋 第二部分：任务分解器

### 2.1 任务树

```
🎯 主目标
│
├── 📌 阶段一：______
│   ├── 任务 1.1 [优先级：P0/P1/P2]
│   ├── 任务 1.2 [优先级：P0/P1/P2]
│   └── 任务 1.3 [优先级：P0/P1/P2]
│
├── 📌 阶段二：______
│   ├── 任务 2.1 [优先级：P0/P1/P2]
│   └── 任务 2.2 [优先级：P0/P1/P2]
│
└── 📌 阶段三：______
    └── ...
```

**优先级定义：**
- **P0**：必须做，不做无法继续（关键路径）
- **P1**：应该做，影响效率/质量
- **P2**：可以做，优化项/可延后

### 2.2 投入方式确认

| 投入类型 | 选项 | 你的选择 |
|----------|------|----------|
| **时间投入** | 全职 / 兼职 (___小时/周) / 碎片时间 | |
| **资金投入** | 无 / 少量 (<___元) / 中等 (___元) / 大量 (___元) | |
| **人力投入** | 独自 / 2-3 人 / 团队 (___人) | |
| **工具/技术** | 现有技能 / 需要学习 / 需要外包 | |

### 2.3 时间估算模型

**单任务估算：**
```
乐观时间 + (4 × 最可能时间) + 悲观时间
期望时间 = ─────────────────────────────
                        6
```

**项目总时间：**
```
总时间 = ∑(关键路径任务期望时间) × 风险系数 (1.2-1.5)
```

## 🚀 使用方式

**每次你要做一件事，告诉我：**

> "我要做 [XXX]，帮我分析"

**我会自动输出：**

1. 📊 思维决策分析（输入输出、要素流动、风险、收益比）
2. 📋 任务分解（任务树、优先级、投入确认、时间估算）
3. ❓ 追问（需要确认的信息、约束条件）
4. ✅ 最终行动计划

## 📝 示例

**输入：** "我想做一个个人财务追踪系统"

**输出：**
1. 分析：涉及数据收集、存储、展示、安全等环节
2. 风险：数据泄露、维护成本、数据源稳定性
3. 收益：财务透明、决策支持、习惯养成
4. 任务：需求分析→技术选型→开发→测试→部署
5. 投入确认：你每周能投入多少时间？有云预算吗？
6. 时间估算：根据投入，预计 4-8 周完成 MVP
