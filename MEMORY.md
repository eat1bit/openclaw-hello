# MEMORY.md - 长期记忆

## 🧠 用户思维模型：芒格式决策分析器

**创建时间**：2026-03-05

### 核心框架

当用户提出要做一件事情时，自动调用以下分析流程：

#### 1. 系统分解
- 分解出事情的各个组成部分
- 介绍每个部分的功能（必须要求）

#### 2. 输入输出分析
- 分析整体系统的输入和输出
- 追踪输入 A 在各个部分如何流动
- 绘制数据/资源流动路径

#### 3. 风险与收益评估
- 列出风险项
- 计算/估算收益比
- 给出风险 - 收益分析

#### 4. 任务分解
- 根据输入 A 流经的各个要素拆分具体任务
- 给出任务优先级（P0/P1/P2...）
- 询问用户投入方式（时间/资金/人力等）
- 询问投入条件
- 根据投入判断完成时间

### 工作模式
- 使用 **plan 模式** 进行对话
- 先完善框架，再执行分析
- 保持结构化输出

## 📋 框架细节（已确认）

### 风险分级
- 🔴 **致命** - 可能导致项目完全失败
- 🟠 **严重** - 重大影响，需要额外资源解决
- 🟡 **一般** - 可管理的影响
- 🟢 **轻微** - 几乎不影响结果

### 收益描述
- **高** - 投入产出比 > 3:1
- **中** - 投入产出比 1.5:1 - 3:1
- **低** - 投入产出比 < 1.5:1

### 任务优先级标注
- **P0** - 必须做，不做无法推进
- **P1** - 重要，影响质量/效率
- **P2** - 可选，优化项
- **标注** - `[并行]` 或 `[串行]` 标记依赖关系

### 时间估算（三场景）
- **乐观** - 一切顺利的最短时间
- **保守** - 正常情况下的时间
- **悲观** - 考虑意外和延迟的时间
